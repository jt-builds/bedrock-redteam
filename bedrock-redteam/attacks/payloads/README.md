# Poisoned policy documents

This directory holds the actual Markdown files that get uploaded to S3 during
attack runs. Each file is a variant of the AnyCompany returns policy with an
indirect prompt-injection payload embedded.

Files are generated by `payloads.py` or can be hand-crafted for exploratory
testing.

## Naming convention

`<payload_family>_returns_policy.md`

Examples:
- `naive_instruction_returns_policy.md`
- `unicode_tag_returns_policy.md`
- `data_exfil_returns_policy.md`
